# Project Triage

## When to use this skill

Reviewing project status, deciding what to work on next, or the weekly
review reaches its dev step.

## Procedure

1. For each active project, answer: shipped anything this week? blocked?
   still worth the slot?
2. Every project keeps exactly one written **next action** small enough for
   a single sitting.
3. Projects with no progress for three consecutive weeks move to the icebox —
   explicitly, with a one-line reason.
4. Cap active projects at three; a new project must displace an old one.

## Boundaries

- Never start a new project during triage — only queue it for consideration.
